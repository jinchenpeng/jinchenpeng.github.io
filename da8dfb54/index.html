mysql实用教程

发表于2020-10-23|更新于2021-04-24
|字数总计:1.9k|阅读时长:7分钟|评论数:

数据库概念

数据和我们的生活息息相关，如何存储数据是个重头戏，现在的数据库产品很多各有千秋，如Mysql、Oracle、sqlserver、db2、postgresql等，他们之间有共同点也有区别

Mysql相关概念

开发中用的最多的应该就是Mysql了，开源免费社区活跃高效存储成为他的标签，首先来了解几个术语

DB：Database

DBMS：Database Management System

SQL：Structure Query Language

DQL：Data Query Language（如select）

DML：Data Manipulation Language（如insert、delete、update）

DDL：Data Define Language（如create、drop、alter）

TCL：Transaction Control Language（如set autocommit=0、start transaction、savepoint、commit、rollback）

数据库：放表的容器

表：放数据的容器

字段：一个表由多个字段组成

记录：表中的一行数据(可以对应一个javabean对象)

Mysql安装

下载：https://dev.mysql.com/downloads/mysql/5.7.html#downloads

安装：http://www.itsoku.com/article/192

Mysql常用命令

  • win+R输入services.msc，找到mysql服务，右键启动或者停止
  • 管理员运行cmd，输入net start mysql开启mysql服务，net stop mysql停止mysql服务
  • mysql -h 主机 -P 端口 -u 用户名 -p 密码，登录本机-h和-P可省略
  • 未登录查看mysql版本mysql —version
  • 登录查看mysql版本select version();
  • 显示所有数据库show databases;
  • 使用指定数据库use 数据库名;
  • 显示当前库中的所有表show tables;
  • 显示其他库的所有表show tables from 数据库名;
  • 查看表的创建语句show create table 表名;
  • 查看表结构desc 表名;

Mysql规范

  • SQL中的关键字最好大写，表名、字段名最好小写
  • 适当的缩进
  • ;是SQL语句结束的标志，多个SQL语句是SQL脚本
  • #是单行注释，— 也是单行注释(中间有个blank)，/*注释*/为多行注释

Mysql常见的数据类型

  • 整数类型：bit、bool、tinyint、smallint、mediumint、int、bigint
  • 浮点类型：float、double、decimal
  • 字符串类型：char、varchar、tinyblob、blob、mediumblob、longblob、tinytext、text、mediumtext、longtext
  • 日期类型：Date、DateTime、TimeStamp、Time、Year

以下详细介绍

整数类型

  • int [(n)] [unsigned] [zerofill] 中括号表示可选的内容
  • bit是位的概念，非0即1
  • 当设一个字段的类型是bool时，他会自动转换为tinyint(1)类型

盗张图

根据二进制的计算规则，无符号能表示的正数的范围肯定是比有符号大的，例如tinyint是用一个字节来存储的，默认有符号的的存储范围为10000000至01111111其中第一位为符号位0代表正数，1代表负数，负数计算数值的时候采用取反加一的补码原理，即-128至127，mysql中的默认为有符号，无符号的需要指定字段为unsigned类型

有的时候，我们发现数据库中的有些字段的类型后面还跟了个数字，如int(n)，这个只要记住两点，首先int是4个字节是不会变的，其次这个n是在数据库中这个字段数据显示的宽度，不足的用0填充(而且需要设定这个字段为zerofill)，超过这个数字时无视宽度直接显示整个数据

我们来测试一下

对比发现默认int的n的为11，bigint默认的n为20，就分析int，设定为unsigned时n变10，但是仅仅是这样还不会用0填充，我们发现设定为zerofill和unsigned zerofill的效果是一样的，设定zerofill自动将int的类型由原来的有符号变为了无符号，且显示的宽度为int所能表示无符号整形的最大宽度(int的无符号最大值为4,294,967,295正好是10个宽度)

插入数据理解一下

b字段插入了-1我们发现数据自动变为了0，说明该字段是不支持有符号的，int(10)和bigint(20)的作用大家可以在这里验证下

浮点类型

类型 字节大小 范围（有符号） 范围（无符号）
float[(m,d)] 4 (-3.402823466E+38,3.402823466351E+38) [0,3.402823466E+38)
double[(m,d)] 8 (-1.7976931348623157E+308,1.7976931348623157E+308) [0,1.7976931348623157E+308)
decimal[(m,d)] m>d为m+2个字节，否则为d+2个字节 依赖于m和d 依赖于m和d

准确来说float和double属于浮点类型，decimal属于定点类型，m是值的宽度(精度)，d是小数点后的长度(标度)，默认float和double不指定时按实际输入显示，decimal默认为decimal(10,0)

插入数据验证一下

纵横对比我们发现，float和double与decimal相比较入舍的规则不太一样，decimal采用的是正常的四舍五入，当输入的小数位超过其标度时会有warning，float和double采用的时四舍六入五成双的规则，待我解释一波

比如test2表中插入1.151和1.152时，标度为1，5是判断位，5后面有数字，直接四舍五入，所以结果都是1.2，而插入1.25和1.35时，5后面没有数字，则根据前面的数字来判断，1.25是2，是偶数，直接舍弃得1.2，1.35是3，是奇数，进一位得1.4

相比较而言float和double容易丢失精度，开发还是用decimal比较合适

TIME 3 ‘-838:59:59’/‘838:59:59’ HH:mm:ss
YEAR 1 1901/2155 YYYY

字符串类型

类型 范围 存储所需字节
char(m) [0,m],m∈[0,2^8-1] m
varchar(m) [0,m],m∈[0,2^16-1] m
tinyblob 0-2^8-1 L+1
blob 0-2^16-1 L+2
mediumblob 0-2^24-1 L+3
longblob 0-2^32-1 L+4
tinytext 0-2^8-1 L+1
text 0-2^16-1 L+2
mediumtext 0-2^24-1 L+3
longtext 0-2^32-1 L+4

char占固定长度，L表示存储的数据本身占用的字节，L 以外所需的额外字节为存放该值的长度所需的字节数，比如tinyblob能够存储0-2^8-1个字节的数据，数据所需要的存储空间是可变的，并且需要1个字节的tinyint整形来记录存储数据所需字节的个数，所以是L+1

数据库数据类型选择习惯

  • 选小不选大
  • 简单易操作
  • 尽量避免NULL
  • 浮点用decimal
  • 时间转整形(时间戳、秒、毫秒)
